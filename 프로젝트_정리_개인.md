### 2. AppHub - Living Portfolio 플랫폼

**프로젝트명**: AppHub - 개인 프로젝트 통합 관리 플랫폼
**기간**: 2025.10 ~ 진행 중
**역할**: 1인 풀스택 개발

**기술스택** (2025년 10월 업데이트):
- Frontend: React 19, Next.js 15, TypeScript
- Backend: FastAPI, Python
- Database: Drizzle ORM, PostgreSQL, Qdrant (Vector DB), Neo4j (Graph DB)
- Authentication: better-auth
- State Management: Zustand
- AI/ML: LangChain, RAG, OpenAI API
- Linting: Biome
- Package Manager: Bun
- Deployment: Vercel, Railway, Fly.io (향후 GCP, 홈서버)

**프로젝트 개요**:
- 기존 정적 포트폴리오를 넘어 **실제 동작하는 프로젝트들을 통합 관리**하는 Living Portfolio 플랫폼
- 모든 개인 프로젝트에서 재사용 가능한 **공통 인프라 플랫폼** 구축
- 인증, 데이터베이스, RAG, 챗봇 등 반복 기능을 하나의 플랫폼으로 통합

#### 핵심 기능:

**1) 공통 인프라 플랫폼**
- Single Sign-On (SSO): 한 번 로그인으로 모든 프로젝트 접근
- 통합 AI 백엔드: 프로젝트 간 데이터 공유 및 AI 에이전트
- 통합 검색: 모든 프로젝트의 데이터를 한 번에 검색
- 마이크로서비스 아키텍처: 확장 가능한 설계

**2) Living Portfolio의 차별점**
- **실증적 기술 역량 증명**: "만들 수 있어요" → "만들어서 운영하고 있어요"
- **즉시 체험**: 면접관이 직접 프로젝트를 사용하고 테스트 가능
- **통합 아키텍처**: 개별 프로젝트마다 별도 인프라 → 통합 플랫폼

**3) AI 서비스 레이어**
- 통합 지능 시스템: 모든 프로젝트의 AI 기능 중앙화
- 대화형 인터페이스: 프로젝트 안내 챗봇 및 실시간 질의응답
- RAG 시스템: 모든 프로젝트에서 공통 사용

**4) 하이브리드 검색 시스템**
- Qdrant (Vector DB): Dense 벡터 검색
- Neo4j (Graph DB): 관계 기반 검색
- pgvector: PostgreSQL 기반 벡터 검색 실험

#### 설계 원칙:
- **관심사의 분리**: AI/ML 로직과 웹 UI 독립 개발
- **확장 가능한 모듈 구조**: 새 프로젝트 쉽게 추가
- **사용자 중심 경험**: 일관된 UI/UX 및 실시간 상호작용

#### 비즈니스 성장 단계:
- **Phase 1 (MVP)**: 콘셉트 검증, 기본 프로젝트 3~5개 통합
- **Phase 2 (확장)**: 기능 다양화, 프로젝트 모듈 다양성
- **Phase 3 (플랫폼화)**: 다른 개발자도 사용 가능한 범용 도구

#### 주요 성과 (예상):
- 최신 기술 스택 경험 (React 19, Next.js 15, better-auth, Drizzle ORM, Biome, Bun)
- Living Portfolio 콘셉트로 차별화된 기술 역량 증명
- 공통 인프라 재사용으로 개발 효율성 극대화

### 3. Men-in-Black - 교통 법규 위반 차량 감지 시스템

**프로젝트명**: 차량 블랙박스 영상 속 도로 교통 법규 위반 차량 감지
**주최**: 새싹 AI 부트캠프
**기간**: 2023.10 ~ 2023.12 (약 3개월)
**역할**: 팀 프로젝트 (4명) - License Plate Recognition 및 Monocular Depth Estimation 담당

**기술스택**:
- Computer Vision: YOLOv8 (n, m), SORT (Object Tracking), EasyOCR
- Depth Estimation: ZoeDepth, MiDaS v3
- ML/DL: PyTorch, TensorFlow, OpenCV
- Backend: FastAPI
- Environment: NVIDIA GPU, AWS, Ubuntu, Anaconda
- Collaboration: GitHub, Notion, Slack, Jira

**프로젝트 개요**:
- 블랙박스 영상에서 교통 법규 위반 차량을 자동으로 감지하고 신고하는 AI 시스템
- 차량 감지, 번호판 인식, 거리 추정 등 종합적인 Computer Vision 기술 적용

#### 담당 업무:

**1) License Plate Recognition (번호판 인식)**

**차량 감지 (Vehicle Detection)**
- Model: YOLOv8n, YOLOv8m
- Dataset: COCO Dataset (330K images, 80 object categories)
- Classes: Car, Motorcycle, Bus, Truck
- Object Tracking: SORT 알고리즘 (실시간 2D 다중 객체 추적)

**번호판 감지 (License Plate Detection)**
- Model: YOLOv8m (50 epoch, 120 epoch)
- Dataset: Roboflow License Plate Recognition (24,242 images)
- Data Augmentation:
  - Flip (Horizontal), Crop (0~15%), Rotation (±10°)
  - Shear (±2°), Grayscale (10%), Hue/Saturation/Brightness/Exposure (±15%)
  - Blur (0.5px), Cutout (5 boxes, 2% size)
- Hyperparameters:
  - Optimizer: SGD (lr=0.01, momentum=0.9)
  - Image size: 640x640
  - Batch size: 16
  - Epochs: 500 (patience=50)

**OCR (Optical Character Recognition)**
- Model: EasyOCR
- Preprocessing:
  - Grayscale 변환 → CLAHE (Contrast Limited Adaptive Histogram Equalization)
  - Gaussian Blur → Canny Edge Detection
  - Contour 찾기 → Perspective Transformation
- 번호판 영역 추출 및 텍스트 인식

**2) Monocular Depth Estimation (단안 거리 추정)**

**ZoeDepth 모델 구현**
- Zero-Shot Transfer by Combining Relative and Metric Depth
- 단일 카메라 영상에서 차량 간 거리 추정
- 실시간 거리 추정 및 안전거리 위반 탐지

#### 프로젝트 구조:
- Line Violation Detection (차선 위반): 진한별
- Traffic Light Recognition (신호등 인식): 최우석
- License Plate Recognition (번호판 인식): 신승엽
- Monocular Depth Estimation (거리 추정): 신승엽

#### 주요 성과:
- YOLOv8 기반 차량 및 번호판 실시간 감지
- EasyOCR을 활용한 번호판 텍스트 인식
- ZoeDepth 기반 차량 간 거리 추정
- 종합적인 Computer Vision 프로젝트 경험

**한계점 및 개선 방향**:
- 블랙박스 영상의 낮은 화질로 인한 OCR 정확도 저하
- 상대 속도가 빠를수록 인식률 감소
- 향후 개선: 다양한 Object Detection 모델 비교, Tesseract OCR/PaddleOCR 실험, Segmentation 시도

**기간**: 2024.09 ~ 현재 (운영 중)
**역할**: 1인 개발 (기획, 개발, 운영)

**기술스택**:
- Platform: 메신저봇R (Android)
- Language: JavaScript (Rhino JavaScript Engine)
- AI: LLM API 연동 (OpenAI GPT 등)
- Integration: Notion API, Web Scraping
- Deployment: 안드로이드 공기계

**프로젝트 개요**:
- 안드로이드 폰에 구현한 카카오톡 기반 개인 비서 LLM 챗봇
- 단체 채팅방에서 즉시 호출 가능한 LLM 서비스
- URL 요약 및 Notion 자동 정리 기능

#### 주요 기능:

**1) LLM 대화 기능**
- 카카오톡 단체 채팅방에서 바로 호출 가능
- 별도 설명 없이 즉시 사용 가능한 직관적 인터페이스
- 메신저봇R 기반 알림 감지 및 자동 응답

**2) URL 요약 및 Notion 정리**
- 채팅방에 공유된 URL 자동 감지
- 웹 페이지 내용 크롤링 및 요약 생성
- 요약 내용을 Notion 페이지에 자동 저장
- 나중에 Notion에서 정리된 내용 확인 가능

**3) 개인 워크플로우 최적화**
- 유용한 URL 발견 시 카톡으로 전송
- 자동으로 요약 및 Notion 정리
- Notion 페이지에서 체계적으로 정리된 내용 검토

#### 기술적 구현:

**1) 메신저봇R 활용**
- Rhino JavaScript 엔진 기반 스크립트 작성
- 카카오톡 알림 감지 및 메시지 파싱
- `response()` 함수를 통한 실시간 메시지 처리
- 채팅방별 컨텍스트 관리

**2) Notion API 연동**
- Notion Database에 요약 내용 자동 저장
- 페이지 생성 및 콘텐츠 포맷팅
- 메타데이터(URL, 날짜, 카테고리 등) 자동 기록

**3) 안드로이드 공기계 운영**
- 듀얼 넘버 카카오톡 계정 사용
- 24/7 상시 운영 환경 구축
- 알림 기반 이벤트 처리

#### 주요 성과:
- 일상적인 정보 수집 및 정리 워크플로우 자동화
- URL 공유 시 즉시 요약 및 Notion 정리로 생산성 향상
- 단체 채팅방에서 LLM 서비스 간편하게 활용
